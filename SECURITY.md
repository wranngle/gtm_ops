# Security Policy

## Reporting a Vulnerability

If you believe you've found a security vulnerability in `wranngle-gtm-engine`, please report it privately. **Do not open a public issue.**

**Preferred:** Use GitHub's Private Vulnerability Reporting:

[Report a vulnerability](https://example.com/REPO_URL_NOT_DETECTED/security/advisories/new)

**Fallback:** Email [bot@gemini.com](mailto:bot@gemini.com).

Please include:

- A description of the issue and its potential impact
- Steps to reproduce (proof-of-concept code, if available)
- The affected version, commit, or branch
- Any suggested mitigation

## What to Expect

This is a personal/demo project maintained on a best-effort basis. Realistic expectations:

- **Acknowledgement:** typically within 7 days
- **Triage and fix:** timing depends on severity and maintainer availability
- **Disclosure:** coordinated with the reporter once a fix is available, or the issue is determined to be out of scope

## Scope

In scope: source code in this repository.

Out of scope: third-party dependencies (please report upstream), issues requiring physical access, social engineering, and findings against forks or deployments not operated by the maintainer.

## Supported Versions

Only the latest commit on the default branch is supported. Older tags and branches do not receive security fixes.

Thank you for helping keep the project and its users safe.
