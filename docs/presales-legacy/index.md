## Executive Summary

The Unified Presales Report is a document generation pipeline that transforms unstructured business intake (job posts, RFPs, client notes) into professional presales deliverables. It produces four synchronized documents from a single input:

1. **AI Process Audit** - Traffic light diagnostic identifying automation opportunities
2. **Project Plan** - Technical scope with milestones and timeline
3. **Commercial Proposal** - Client-facing pricing with ROI analysis
4. **Internal Cost Sheet** - Margin analysis and negotiation guardrails

The system integrates intelligent technical research to provide accurate project cost estimations based on real tool accuracy and n8n workflow complexity.

## Core Architecture

### Entry Points

| Entry | Purpose | Command |
|-------|---------|---------|
| `cli.js` | CLI interface | `wranngle generate input/file.txt` |
| `server.js` | Web dashboard + API | `npm start` → http://localhost:3000 |
| `index.js` | Library exports | `import { ... } from './index.js'` |

### Pipeline Flow

```
Input (txt) → Extract → Research → Estimate → Transform → Render → Polish → PDF
```

**Detailed Flow:**

1. **Extract** (`lib/extract.js`)
   - LLM parses unstructured text into structured intake schema
   - Extracts: client info, workflow definition, systems, pain points, volumes

2. **Research** (`lib/integration_research.js` + `lib/proactive_research.js`)
   - Checks n8n methodology research library cache
   - Falls back to embedded n8n native node database (60+ integrations)
   - LLM enhancement for unknown integrations
   - Generates research gap reports with actionable commands

3. **Estimate** (`lib/estimate.js`)
   - Uses research-derived base hours
   - LLM generates role-based effort breakdown
   - Cost calculated from hours × agency rates

4. **Transform** (`lib/pipeline.js` + `lib/template_context.js`)
   - Builds document schemas for each output type
   - Syncs bleed/savings data across documents
   - Generates _display fields for Mustache templates

5. **Render** (`templates/presales_report.html`)
   - Mustache templating with 4-sheet layout
   - Unified header/footer components
   - Brand-consistent styling (Wranngle desert theme)

6. **Polish** (`lib/html_polish.js`)
   - LLM refines narratives and prose sections
   - Ensures professional language and coherence

7. **PDF** (`lib/pdf_generator.js`)
   - Puppeteer generates multi-page PDF
   - Consistent print styling

## Technical Research Integration

### User Priority: Intelligent Cost Estimations

The research system is designed to provide accurate project estimates based on:

1. **Real Tool Accuracy** - n8n native node availability, API quality, auth complexity
2. **Nuanced Notes** - Gotchas, rate limits, client requirements
3. **Authoritative Sources** - API documentation URLs, integration guides

### Data Flow

```
Intake Systems → Research Lookup → Proactive Research → Estimate Enhancement
       ↓                ↓                  ↓                     ↓
section_c.q10    library-index.json   N8N_NATIVE_NODES      enrichedResearch
```

### Research Library Structure

```
n8n_workflow_development/context/technical-research/
├── library-index.json       # Index of all research
├── {slug}.md                # Detailed markdown reports
└── {slug}.json              # Cached JSON for quick lookup
```

### Key Integration Points

| File | Responsibility |
|------|----------------|
| `lib/integration_research.js` | Parse cached research, freshness scoring |
| `lib/proactive_research.js` | n8n node database, LLM fallback research |
| `lib/research.js` | Tier assessment, research queries |
| `lib/build_technical_approach.js` | Build SOW technical section |

### n8n Native Node Database

The `proactive_research.js` contains an embedded database of 60+ integrations with:

- Node package name (e.g., `n8n-nodes-base.hubspot`)
- Authentication type (oauth2, api_key, basic)
- API quality rating (excellent, good, fair, poor)
- Base complexity score (1-10)

This enables **zero-latency lookups** for common integrations without LLM calls.

## Known Issues & Technical Debt

### From REFACTOR.md

1. **$10M Calculation Bug** - Bleed calculation used 5196 minutes instead of hours
   - Root cause: No input validation on `minutes_per_item`
   - Fix planned: Zod schemas with sanity bounds

2. **Display Field Sync** - Numeric values and `_display` fields can desync
   - Must regenerate `_display` after any value recalculation
   - Verification: `npm run verify`

3. **TypeScript Migration** - Planned but not executed
   - Current: 28 JS files with implicit types
   - Target: ~14 TS files with Zod validation

### Research Integration Gaps

1. **Freshness Decay** - Research older than 30 days marked stale
2. **Category Matching** - Generic terms (e.g., "CRM") default to first provider
3. **Missing Citations** - Auto-generated research lacks authoritative sources
