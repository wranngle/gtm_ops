# Core Beliefs

Status: Active
Owner: wranngle
Last reviewed: 2026-04-30

## Beliefs

1. Humans steer; agents execute.
2. The repo is the memory. If an agent cannot see it here, it effectively does not exist.
3. AGENTS.md is a map, not a manual.
4. Fast work requires strict boundaries, not loose prompts.
5. Public work must be synthetic, clean-room, and safe from commit one.
6. Boring, inspectable tools beat opaque abstractions when agent legibility matters.
7. Every recurring taste preference should become either a doc rule or a check.
8. Technical debt should be garbage-collected continuously in small changes.
9. Verification must be local, repeatable, and cheap enough for agents to run often.
10. When an agent struggles, improve the environment instead of asking it to try harder.

## Consequences

- Add fixtures before adding external integrations.
- Add validators before relying on conventions.
- Add execution plans before multi-file changes.
- Keep private operational repos out of public history.
